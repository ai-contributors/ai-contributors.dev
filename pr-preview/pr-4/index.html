§ 01 · Install

Three skills. One audit loop.

Three skills installed via the skills CLI. Profile your repo, run the audit, optionally apply fixes, then loop until you hit your target level. 

# install all three skills from the ai-contributors registry
$ npx skills add ai-contributors/ai-contributor-spec \
    --skill ai-contributor-audit-profile \
    --skill ai-contributor-audit \
    --skill ai-contributor-audit-fix

  ✓ ai-contributor-audit-profile  (optional)  describes the repo
  ✓ ai-contributor-audit                       runs the audit loop
  ✓ ai-contributor-audit-fix      (optional)  applies improvements

→ next: invoke "ai-contributor-audit" from your agent
# pull the latest audit prompts + spec data
$ npx skills update ai-contributor-audit

  ↻ ai-contributor-audit  v0.3.1 → v0.4.0
    · spec rev v0.1 → v0.2 (+3 clauses)
    · evidence collectors updated for monorepos
    · stamp format unchanged — prior audits remain valid
# from any agent that supports skills
  use skill: ai-contributor-audit-profile      # optional, run once
  use skill: ai-contributor-audit              # produces audit.md + evidence.json
  # — read the report, decide what to fix —
  use skill: ai-contributor-audit-fix          # optional, applies suggested edits
  use skill: ai-contributor-audit              # loop back to re-score
requires the skills CLI · node ≥ 20 · skills.sh ↗
  1. optional ai-contributor-audit-profile
    Profile
    Describes the repo so the audit knows what it's looking at — stack, surfaces, deployment shape.
  2. required ai-contributor-audit
    Audit
    Runs the full audit pipeline. Collects evidence, stamps initial findings, hosts auditor judgments, validates the final report.
  3. optional ai-contributor-audit-fix
    Fix
    Applies suggested improvements from the audit. Loops back into the audit skill to re-score.
§ 02 · The audit loop

How the audit skill works.

The audit skill is a pipeline, not a single shot. It collects evidence, stamps an initial pass, hosts auditor judgments, runs a final pass, then validates the artifact — producing a reviewable report and a re-runnable loop.

§ 03 · Levels

Five levels of AI autonomy.

Pick the minimum level your repo conforms to. The level a repository targets is determined by the highest AI workflow it permits, not by the average.

Min. level Name Autonomy Use this when…
L0 Baseline Hygiene 0% AI is not part of the contribution workflow yet. People may still use AI for personal help, but no AI tool reads the repository as workspace context, creates commits, or opens pull requests.
L1 Hardened 25% AI tools may read repository context, explain code, suggest commands, or help with review, but do not produce shippable changes
L2 AI Assisted 50% AI may produce code, docs, tests, configuration, or migrations that a human actively accepts. Extra risks such as tool use, external GitHub content, fetched URLs, retained context, provider routing, and dependency suggestions must be controlled.
L3 AI Authored 75% AI may complete delegated implementation tasks, open pull requests, or make material repository changes for human review
L4 AI Autonomous 100% AI may merge, release, deploy, schedule recurring changes, approve workflows, rotate settings, or otherwise act without per-change human approval

§ 04 · Pillars

Seven pillars, twenty-nine clauses.

The spec is organised into seven pillars. Each one bundles a contiguous range of clauses that together govern one slice of an agent-friendly repo.

§1–4 🏗️

Engineering Foundation

The reproducible environment, static correctness, architecture boundaries, and pre-commit / CI gates that make any change reviewable.

AI Agents

Oversight
