The AI Contributor Specification.

A vendor-neutral specification for what an AI-friendly codebase looks like — what humans owe agents, what agents owe humans, and how a third party audits the contract. These docs are the reference; the marketing site is elsewhere.

§ 01What these docs cover

This site is the technical entry point for adopters. It does not argue for the spec — that's the landing page's job. It assumes you've decided to read the spec, and points you at the right surface for the question you have.

Note

The normative artifact is the specification. Everything in these docs renders content from ai-contributors/ai-contributor-spec; this site does not own any normative text.

§ 03How to read these docs

Pick the entry that matches what you're trying to do.

  • Auditing a repo right now. Open the no-install prompt, paste it into your agent, point it at the working tree.
  • Adopting on a new repo. Read the TypeScript + pnpm guide if your stack matches; otherwise read the specification directly and translate.
  • Understanding the audit machinery. The evidence model explains how findings are produced and what makes the result deterministic.
  • Building tooling against the spec. The repository is the source of truth. Pin a spec rev; everything in this site renders from a pinned submodule (currently 9d6e067).

§ 04Versioning & stability

The site renders an immutable copy of the upstream spec, pinned by SHA. The current pin is 9d6e067 on tag v0.1. To trust a particular reading, pin to that revision; to follow upstream, the site rebuilds when the submodule pointer moves.

§ 05Contributing

Issues, pull requests, and audits of real repos belong in ai-contributors/ai-contributor-spec. This site only renders. Wording, restructuring, and new sections all belong upstream.
